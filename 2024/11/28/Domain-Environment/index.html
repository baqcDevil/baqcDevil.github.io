搭建域环境

为了更好的了解内网渗透，搭建域环境进行实操

基础知识

组

组：是用户的集合，由多个用户账户组成，组名可以看成组成员共同的名字。组是为了简化管理，可以一次性对组进行授权，实质上就是给该组的所有用户同时进行授权。注意一个用户可以同时属于多个组，用户对某个资源的权限等于它在各组的权限之和，但是如果一旦有个组对某个资源的权限为禁止，那么该用户将不得访问该资源，因为禁止是优先的

组的特点：计算机的地位都是平等的。每一台计算机都独立的维护自己的资源，不需要集中管理网络资源，每一台计算机都在本地存储用户的账号，本地账号只能登陆本地计算机

域

域是将多台计算机在逻辑上组织到一起，进行集中管理，也就是创建在域控制器上的组，将组的账户信息保存在活动目录，域组可以用来控制域内任何一台计算机资源的访问和执行系统任务的权限

域是组织和存储资源的核心管理单元，域的核心是域控制器，与控制器的核心是活动目录，活动目录相当于一个数据库

活动目录

提供的目录服务，在中央数据库存放信息，使用户在网络上只拥有一个用户账号。活动目录主要包括目录和目录服务，目录是存储各种对象的一个物理容器，目录服务是使目录中的信息资源发挥作用的服务。活动目录可以管理诸如计算机对象、用户账号、打印机之类的网络资源。活动目录是一个分布式的目录服务，信息可以分散在多个不同的计算机上，保证用户能够快速访问

域环境搭建

前期准备

虚拟机：VMware Workstation 17 Pro

操作系统：Windows Server 2016（域控）、Windows10 x64

实验步骤

安装操作系统，Windows Server 2016使用 Administrator 账号登录系统

设置静态IP地址

域控的ip必须是静态的，下面是设置静态IP步骤

2016的ip：192.168.150.133

10的ip：192.168.150.134

安装活动目录和DNS服务器

在安装前，先给该计算机重命名为 DC
重启后点击添加角色和功能

默认下一步，选择域服务

默认下一步安装

升级域控

点击黄色的旗子，然后点击将此服务器提升为域控制器

选择添加新林，设置根域名为 test.local ，这里最好不要填外网有的域名，不然很容易出错。

输入还原模式密码，这个密码可以与 Administrator 账号密码不一致，这里我输入的是一样的

默认下一步，安装

安装完后会重启，重启后，在输入账号处，可以看到账号名称发生了变化，变成了 TEST\Administrator ，这是域账号的登录模式，要登录一个域，需要在账号前面加上域名称和斜杠。输入密码登录系统。如果要登录本地账号，则需要输入.\administrator 来登录

我们再次查看下IP，可以看到首选 DNS 服务器已经设置成 127.0.0.1 ，也就是域控制器自己，如果需要上网，可以把备用DNS服务器设置成之前的值
