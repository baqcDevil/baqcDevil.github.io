解决Git连接失败：Failed to connect to github.com port 443 after 21090 ms: Couldn‘t connect to server ‍

通常发生在使用VPN后，系统端口号与Git端口号不一致时

今天上传笔记至博客遇到这个问题，搜索后知道代理的问题

首先，确认是否在使用VPN。VPN的使用可能会改变本机的系统端口号，从而影响到Git的正常连接

解决办法：配置http代理Windows、Linux、Mac OS 中 git 命令相同：
配置socks5代理

配置http代理

socks5和http两种协议由使用的代理软件决定，不同软件对这两种协议的支持有差异，如果不确定可以都尝试一下

总结：

问题场景 解决方法 重要命令
使用VPN时 调整Git代理设置 git config –global http.proxy, git config –global https.proxy
未使用VPN时 取消Git代理设置 git config –global –unset http.proxy, git config –global –unset https.proxy
